#  系统数据流说明


## 数据传输流程
```
    ADC采样
        |
    DMA搬运
        | 
    数据处理
        |  
    帧协议封装 
        |
    UART发送
        |
    UART DMA接收
        | 
    帧解析
        |  
    CRC校验
        |     
    数据恢复
        |   
    应用输出
```
##  TX（发送端）流程

1. ADC采集四路摇杆数据
2. DMA搬运到 adc_buf
3. 通过滤波 +  死区处理调整 adc_buf 值
4. 将调整后新数据封装为Frame结构体
5. 计算CRC-16值
6. UART发送


## RX（接收端）流程

1. 在标准CRC-16校验下通过UART DMA接收数据
2. 接收值存进dma_rx_buf
3. 在字节流中核对帧头，传递完整帧
4. 恢复控制通道值
5. PC13指示灯输出验证(暂时使用此验证，后续会持续优化)


##  当前系统数据
- 通过协议层数据传输
- DMA + UART 稳定发送/接收
- 自定义Frame协议
- 基础帧解析与数据恢复完成
- 已优化为工程CRC-16校验
- 暂不支持(LoRa/ELRS)等，后续会修改优化
